# RESP (Redis Serialization Protocol)

RESP:

Redis Serialization Protocol

Redis client va Redis server orasidagi aloqa protokoli.

---

# Nega RESP Kerak?

RESP:

ma'lumotlarni
aniq va ishonchli parse qilish

uchun kerak.

Oddiy text parser:

SET bio I love Golang

kabi holatlarda adashishi mumkin.

RESP esa:

har bir qiymat uzunligini
ham yuboradi

---

# RESP Types

Asosiy RESP turlari:

Simple String
Bulk String
Array
Integer
Error
Null

---

# RESP Array

RESP Array:

*<count>

ko'rinishida bo'ladi.

Misol:

*3

Ma'nosi:

3 ta element keladi

---

# Array Misoli

*3
$3
SET
$4
name
$5
adham

Bu:

SET
name
adham

degan commandni bildiradi.

---

# Bulk String

Bulk String:

$<length>

ko'rinishida bo'ladi.

Misol:

$4
PING

Bu:

uzunligi 4 byte
bo'lgan string

degani.

---

# Nega Length Yuboriladi?

Length:

string nechta byte ekanini
aniq bilish

uchun yuboriladi.

Misol:

hello world

RESP:

$11
hello world

Parser:

aniq 11 byte o'qiydi

# RESP Parser

RESP Parser:

RESP requestni
Go arrayga aylantiradi

Misol:

Input:

*3
$3
SET
$4
name
$5
adham

Output:

[]string{
	"SET",
	"name",
	"adham",
}

---

# RESP Encoder

RESP Encoder:

server javoblarini
RESP formatga o'tkazadi

Misol:

OK

↓

+OK\r\n

---

# CRLF

RESP standartida:

\r\n

ishlatiladi.

Bu:

Carriage Return
Line Feed

degani.

RESP'dagi har bir qator:

\r\n

bilan tugaydi.

---

# Byte-Level Parsing

Oldingi parser:

strings.Fields()

asosida ishlaydi.

RESP Parser esa:

byte darajasida ishlaydi

Misol:

*3
$3
SET
$4
name
$5
adham

baytma-bayt o'qiladi.
