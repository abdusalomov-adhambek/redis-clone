# Redis Clone — 12-ETAP Nazariyasi

# DEL Command

DEL:

Redis'dagi keyni o'chirish

uchun ishlatiladi.

Misol:

DEL name

---

# DEL Maqsadi

Storage ichidagi ma'lumotni:

qo'lda o'chirish

imkoniyatini beradi.

---

# DEL Misoli

SET name Adham

↓

GET name

↓

Adham

---

Keyin:

DEL name

↓

GET name

↓

NULL

---

# Expiration va DEL Farqi

### Expiration

TTL tugaganda
avtomatik o'chadi

---

### DEL

foydalanuvchi
qo'lda o'chiradi

---

# Manual Deletion

Manual Deletion:

foydalanuvchi tomonidan
qo'lda o'chirish

jarayoni.

DEL command bunga misol bo'ladi.

---

# Storage Delete

DEL ishlaganda:

delete(storage, key)

bajariladi.

Natija:

key va value
storage'dan o'chadi

---

# Expiration Delete

Agar key uchun TTL mavjud bo'lsa:

delete(expirations, key)

ham bajarilishi kerak.

---

# Nega Expiration Ham O'chiriladi?

Faraz qil:

Storage:
name -> Adham

---

Expirations:
name -> expireAt

---

Agar faqat storage'dan o'chirilsa:

Storage:
empty

---

Expirations:
name -> expireAt

qolib ketadi.

---

Natija:

storage va expirations
bir-biriga mos kelmay qoladi

---

# Consistency

Consistency:

bog'langan ma'lumotlar
bir xil holatda bo'lishi

demakdir.

Bizda:

Storage
+
Expirations

har doim mos bo'lishi kerak.

---

# Redis Response

qaytaradi.

---

# Integer Response

Integer Response:

:<number>

ko'rinishida bo'ladi.

Misol:

:1

yoki

:0

---
